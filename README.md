## STEP 1. 주제 설정 및 데이터 선정

> �� 큰 주제 → 데이터 탐색 → 주제 구체화 순서로 진행하세요. 주제 선정에 너무 많은 시간을 쓰지 않는 것이 핵심입니다.
> 

### 1-1. 큰 주제 브레인스토밍

- [ ]  팀원 각자 관심 도메인을 슬랙에 자유롭게 포스팅
- [ ]  슬랙 이모지 반응으로 1차 투표 → 큰 주제 1~3개로 압축
- [ ]  줌 회의에서 주제 후보 토론 및 최종 방향 결정
- [ ]  결정된 주제 후보와 근거를 슬랙에 정리하여 공유
- [ ]  해당 주제의 공개 데이터 존재 여부 확인 — 크롤링, 공공데이터포털, Kaggle, AI Hub 등

### 1-2. 좋은 주제 vs 나쁜 주제

> �� 주제는 **좁고 구체적일수록** 분석이 명확해지고 인사이트가 날카로워집니다.
> 

| 구분 | 나쁜 주제 ❌ | 좋은 주제 ✅ |
| --- | --- | --- |
| 너무 넓음 | 소비 트렌드 분석 | 20대 여성의 뷰티 앱 내 야간 구매 패턴 분석 |
| 검증 불가 | 고객이 원하는 것 파악 | 리뷰 평점 3점 이하 고객의 재구매율과 이탈 시점 분석 |
| 데이터 없음 | 미래 매출 예측 | 최근 2년 계절별 매출 변동과 프로모션 효과 분석 |
| 결론 뻔함 | 연령별 소득 차이 분석 | 같은 소득대 30·40대의 지출 카테고리 차이 분석 |
| 너무 기술적 | 머신러닝으로 고객 분류 | 구매 주기와 객단가 기준으로 고객 4개 그룹 분류 및 특징 비교 |
|  |  |  |
|  |  |  |

**주제 좁히기 예시**

```
1단계 (너무 넓음):  "배달 앱 분석"
2단계 (조금 좁힘):  "배달 앱 사용자 행동 분석"
3단계 (더 좁힘):   "배달 앱 재주문율에 영향을 주는 요인 분석"
4단계 (적절함):    "배달 앱에서 첫 주문 후 7일 내 재주문 여부에 영향을 주는
                   리뷰·배달시간·할인쿠폰 요인 분석"
```

- [ ]  주제가 위 4단계 수준으로 좁혀졌는지 확인
- [ ]  줌 회의에서 팀원들이 주제를 듣고 "무엇을 분석하는지" 즉시 이해하는지 확인

### 1-3. 데이터 선정

- [ ]  팀원 각자 찾은 데이터셋을 슬랙에 링크 + 간략 설명과 함께 공유
- [ ]  줌 회의에서 데이터셋 검토 및 최종 선정
- [ ]  선정된 데이터 명세를 슬랙에 정리하여 공유
    - 출처 및 원본 링크, 파일 형태, 수집 기간, 행/열 수, 주요 컬럼 설명
- [ ]  데이터 라이선스·저작권 확인 후 슬랙에 공유
- [ ]  원본 데이터 파일을 팀 공유 폴더(Google Drive 또는 GitHub)에 저장 후 슬랙과 노션에 링크 공유

### 1-4. 초기 주제 가설 수립

- [ ]  슬랙에서 가설 후보 자유 제안 및 의견 교환
- [ ]  줌 회의에서 가설 검토 및 최종 2~5개 확정
- [ ]  확정된 주제와 가설을 슬랙과 노션에 기록
- [ ]  줌 회의 요약을 슬랙과 노션에 공유

## STEP 2. 기본 EDA (탐색적 데이터 분석)

> �� 그래프와 교차표를 최대한 많이 그려보세요. 각 유형별 최소 10개 이상이 목표입니다. 각 그래프마다 인사이트 코멘트를 반드시 작성하세요.
> 

### 2-1. 데이터 전처리

- [ ]  결측값 현황 파악 및 처리 방식 결정 — 제거 / 대체(평균·중앙값·최빈값) / 유지
- [ ]  중복 데이터 확인 및 제거
- [ ]  데이터 타입 변환 — 날짜 컬럼 → datetime, 숫자형 컬럼 정비
- [ ]  이상값(Outlier) 탐지 — 박스플롯, IQR 기준으로 확인
- [ ]  파생 변수 생성 (필요 시) — 연령대, 매출 구간, 요일 등
- [ ]  전처리 기준과 결과를 슬랙에 공유
- [ ]  전처리 방식에 이견이 있으면 줌 미니 회의 또는 슬랙 스레드로 빠르게 합의

### 2-2. 단변량 분석 (10개 이상)

- [ ]  수치형 변수 히스토그램 — 분포, 정규성 여부, 편향 방향
- [ ]  수치형 변수 박스플롯 — 이상값 및 분포 범위
- [ ]  범주형 변수 막대그래프 — 카테고리별 빈도
- [ ]  기술통계 요약표 — mean, median, std, min, max, 분위수
- [ ]  시계열 데이터의 경우 시간별 추이 그래프
- [ ]  분석 중 막히는 부분은 슬랙에 스크린샷 + 코드 스니펫 공유

### 2-3. 다변량 분석 (10개 이상)

- [ ]  수치형 변수 상관관계 히트맵 — 상관계수 0.5 이상인 쌍 메모
- [ ]  범주형 × 수치형 교차 박스플롯
- [ ]  범주형 × 범주형 교차표(Crosstab) — 비율·건수 모두 확인
- [ ]  피벗 테이블 활용 그룹별 집계
- [ ]  산점도 — 주요 수치형 변수 쌍 조합
- [ ]  각 그래프에 인사이트 코멘트 1줄 이상 기재

### 2-4. 좋은 분석 코멘트 vs 나쁜 분석 코멘트

> �� 그래프를 그리는 것보다 **무엇을 읽어냈는지**가 더 중요합니다.
> 

| 구분 | 나쁜 코멘트 ❌ | 좋은 코멘트 ✅ |
| --- | --- | --- |
| 그래프 설명만 함 | "막대그래프를 그렸다" | "30대 구매 건수가 20대보다 42% 높으며, 특히 주말 오후 집중도가 뚜렷하다" |
| 수치 나열 | "평균 3.2, 중앙값 2.9이다" | "평균(3.2)이 중앙값(2.9)보다 높아 고액 구매자가 평균을 끌어올리고 있음을 시사한다" |
| 결론 없음 | "계절별로 매출이 다르다" | "12월 매출이 연평균 대비 38% 급증하며, 이는 연말 선물 수요와 연관성이 높다" |
| 인사이트 부재 | "상관계수가 0.72다" | "리뷰 수와 재구매율의 상관계수가 0.72로 강한 양의 관계 — 리뷰 유도 전략이 재구매에 실질적 효과가 있을 수 있다" |

### 2-5. EDA 결과 정리

- [ ]  주제와 연관성 높은 분석 결과 상위 5개 선별
- [ ]  예상치 못한 인사이트 슬랙에 공유 — 팀원 의견 수렴
- [ ]  추가로 필요한 데이터 목록 슬랙에 공유
- [ ]  EDA 결과를 줌 회의에서 팀 전체에 발표 및 피드백 수렴
- [ ]  회의 후 주요 피드백과 다음 방향을 슬랙에 요약 공유

## STEP 3. 주제 확정 및 심화 분석

> �� EDA 결과를 바탕으로 주제를 좁혀 잡으세요. 주제는 좁을수록, 검증 가능할수록 좋습니다.
> 

### 3-1. 주제 확정

- [ ]  EDA 인사이트 기반으로 주제 재검토 — 기존 가설과 비교, 보정 또는 교체
- [ ]  주제 범위 구체화
- [ ]  분석 목적·대상·기간 명확화
- [ ]  줌 회의에서 팀 전원 합의 후 최종 주제 확정
- [ ]  확정된 주제를 슬랙에 공지

### 3-2. 심화 분석

- [ ]  핵심 질문 3~5개를 슬랙에 공유하며 팀원 의견 수렴
- [ ]  그룹별 비교 분석 — 세그멘테이션 기준 정의 후 집단 간 차이 검증
- [ ]  시계열 분석 (해당 시) — 추세·계절성·이상 구간 탐지
- [ ]  상관/회귀 분석 (필요 시) — 변수 간 관계 수치화
- [ ]  각 인사이트에 비즈니스 의미 해석 추가
- [ ]  비즈니스 액션 아이디어 초안 슬랙에 공유
- [ ]  심화 분석 결과를 줌 회의에서 발표 및 팀 피드백 수렴

## STEP 5. 발표 준비 및 최종 점검

### 5-1. 슬라이드 구성 원칙

- [ ]  간지(섹션 구분 슬라이드) 삽입 — 챕터가 바뀔 때마다 흐름 구분
- [ ]  슬라이드당 텍스트 최소화 — 핵심 키워드·수치 중심, 문장 나열 금지
- [ ]  그래프는 제목 + 핵심 수치 + 한 줄 해석 함께 배치
- [ ]  분석 결과 그래프는 필요한 만큼 포함 — 슬라이드당 그래프 1~2개 권장
- [ ]  강조할 수치·키워드는 크게, 부가 설명은 작게 — 시각적 위계 구성

### 5-2. 데이터 신뢰도 슬라이드

- [ ]  **데이터 개요 슬라이드** 작성
    - 데이터 출처 및 수집 방법
    - 수집 기간 (예: 2022년 1월 ~ 2023년 12월)
    - 분석 대상 (예: 서울시 거주 20~40대 직장인)
    - 전체 행(레코드) 수 및 열(컬럼) 수
    - 주요 컬럼명과 의미 간략 설명
- [ ]  **전처리 요약 슬라이드** 작성
    - 결측값 처리 방법과 비율
    - 이상값 처리 기준 및 제거 건수
    - 분석에 사용된 최종 데이터 행/열 수
    - 파생 변수 생성 내역
- [ ]  데이터 한계점 1~2줄 언급

### 5-3. 발표 자료 스토리라인

- [ ]  표지 — 팀명, 주제, 날짜, 팀원
- [ ]  목차 슬라이드
- [ ]  **[간지] 문제 정의**
- [ ]  분석 배경 및 목적
- [ ]  **[간지] 데이터 소개**
- [ ]  데이터 개요 슬라이드
- [ ]  전처리 요약 슬라이드
- [ ]  **[간지] 분석 결과**
- [ ]  핵심 질문별 그래프 + 해석 슬라이드 (필요한 만큼)
- [ ]  **[간지] 결론 및 비즈니스 액션플랜**
- [ ]  인사이트 요약 — 핵심 발견 3~5개
- [ ]  **비즈니스 액션플랜 슬라이드**
    - 인사이트 → 제안 액션 → 기대 효과(KPI) 흐름으로 작성
    - 단기(즉시 실행) / 중장기(추가 검토 필요) 구분
    - 실행 주체 명시
- [ ]  마무리 슬라이드 — Q&A 안내

### 5-4. 좋은 비즈니스 인사이트 vs 나쁜 비즈니스 인사이트

> �� 인사이트는 "그래서 어떻게 할 것인가"로 이어져야 합니다. 데이터 사실의 나열로 끝나면 안 됩니다.
> 

**① 인사이트 표현 수준 비교**

| 구분 | 나쁜 인사이트 ❌ | 좋은 인사이트 ✅ |
| --- | --- | --- |
| 사실 나열 | "20대 구매 비율이 35%다" | "20대가 전체 구매의 35%를 차지하지만, 객단가는 40대의 절반 수준 — 볼륨은 크지만 수익 기여도는 낮다" |
| 원인 없음 | "12월에 매출이 높다" | "12월 매출이 평월 대비 38% 높으며, 선물세트 카테고리가 전체 증가분의 61%를 차지 — 연말 선물 수요가 핵심 드라이버" |
| 방향 없음 | "재구매율이 낮다" | "첫 구매 후 30일 내 재구매율이 12%로 업계 평균(22%) 대비 낮음 — 첫 구매 직후 리텐션 개입이 가장 효과적인 구간" |
| 인과 혼동 | "광고비를 늘리면 매출이 오른다" | "광고 집행 주간의 신규 고객 유입이 평균 23% 증가 — 단, 재구매율과는 유의미한 상관관계 없어 광고는 신규 유입에만 효과적" |

**② 액션플랜 연결 수준 비교**

| 구분 | 나쁜 액션플랜 ❌ | 좋은 액션플랜 ✅ |
| --- | --- | --- |
